# Vanchor Helm Board Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A complete, fab-ready KiCad 10 project for the Vanchor Helm Board — Pi 4/5 carrier + Pico 2 I²C slave controlling an external 1500 W speed controller (digipot hijack + reversing contactor) and an onboard-driven 12 V servo (IBT-2 module, AS5600 feedback), all UARTs broken out, 12–48 V input.

**Architecture:** Hierarchical schematic (root + 5 sheets: power, pi, mcu, thrust, servo) hand-authored as `.kicad_sch` s-expressions; PCB generated by a Python script running against the container's `pcbnew` (footprint load, net assign, placement, pours), then routed (scripted power routing + freerouting for signals, fallback scripted manhattan), validated with `kicad-cli` ERC/DRC.

**Tech Stack:** KiCad 10.0.3 in the `vanchor-kicad` docker container (`docker exec vanchor-kicad kicad-cli …`; project visible at `/config/vanchor-pcb`), Python 3 + pcbnew inside the container.

## Global Constraints

- All files live under `/home/alex/projects/vanchor-pcb` (editing outside the project is prohibited; vanchor-ng is read-only reference).
- KiCad file format: 10.x (`(version 20250114)` era as emitted by the container's KiCad; when in doubt, generate a scratch file with the GUI-less CLI or copy headers from a `kicad-cli`-created file).
- Verification gates (the "tests" of this plan): `docker exec vanchor-kicad kicad-cli sch erc --exit-code-violations --severity-error …` and `… pcb drc --exit-code-violations --severity-error …` must exit 0.
- Commit after every task with a `feat:`/`chore:` message + Claude trailer.
- Hardware "test-first" adaptation: for each sheet, the ERC run BEFORE authoring must fail (file absent/unwired), then pass after — recorded in the task log. Full code blocks for `.kicad_sch` are impractical in-plan; the **net tables below are the authoritative content** and the sheet must implement them exactly.

### Thrust sheet (`thrust.kicad_sch`)
- U2 = 74HCT125 DIP-14, VCC=+5V (C6 100n). 1OE/2OE/3OE/4OE (1,4,10,13)=GND. 1A(2)=DP_INC_3→1Y(3)=DP_INC_5; 2A(5)=DP_UD_3→2Y(6)=DP_UD_5; 3A(9)=DP_CS_3→3Y(8)=DP_CS_5; 4A(12)=THR_PWM_3→4Y(11)=THR_PWM_5.
- U3 = X9C103 DIP-8: 1=INC←DP_INC_5, 2=U/D←DP_UD_5, 7=CS̄←DP_CS_5 + R14 10k pull-up to +5V, 8=VCC=+5V (C7 100n), 4=VSS=GND, 3=VH→J13.1, 5=RW→JP1.A, 6=VL→J13.3.
- PWM-DAC: THR_PWM_5 →R15 1k→ node THR_ANA (C8 10 µF→GND) → U4A MCP6002 follower (V+=+5V C9 100n, inA+=THR_ANA, inA−=outA, outA→JP1.B). U4B tied off (inB+=GND, inB−=outB).
- JP1 = 3-pad solder jumper: A=RW, B=outA, C(common)→J13.2 (WIPER) + C10 100n & C11 100 µF →GND.
- J13 = screw 3P "KNOB": 1=VH, 2=WIPER, 3=VL.
- Contactor: Q2 IRLZ44N TO-220: G←R16 100Ω←CONT_GATE_3 + R17 100k→GND; D→J14.2; S→GND. J14 = screw 2P "CONTACTOR": 1=+12V, 2=Q2.D. D7 1N4007: A=J14.2, K=J14.1.

### Servo sheet (`servo.kicad_sch`)
- J15 = 1×8 pin header "IBT-2" (module mounts off-board, cabled): 1=RPWM, 2=LPWM, 3=R_EN, 4=L_EN, 5=R_IS→R18 10k→ISENSE_R (D8 BAT54S clamp +3V3, C12 100n), 6=L_IS→R19 10k→ISENSE_L (D9 clamp, C13 100n), 7=+5V, 8=GND.
- J11 = JST-XH 4P "AS5600": 1=+3V3, 2=GND, 3←R20 100Ω←ENC_SDA, 4←R21 100Ω←ENC_SCL. D10,D11 BAT54S ESD near connector.
